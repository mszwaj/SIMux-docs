Flashing Firmware

How to update SIMux firmware using pre-built binaries.

Overview

Flashing firmware updates the program running on the SAMD21 microcontroller without requiring source code compilation.

When to Flash:

  • Install latest firmware release

  • Recover from firmware corruption

  • Switch between firmware versions

  • Test different firmware builds

What You Need:

  • SIMux board with MCU installed

  • USB-C cable

  • BOSSA flashing tool

  • Firmware binary (.bin file)

Obtaining Firmware Binaries

Official Releases

Location: GitHub Releases

File Format: `SIMux_vX.Y.Z_.bin

Examples:

  • SIMux_v{{ fw_version }}_samd21_debug.bin - Debug build

  • SIMux_v{{ fw_version }}_samd21_release.bin - Release build (optimized)

Recommendation: Use release builds for production, debug builds for development.

Bootloader Mode

SAMD21 has a built-in UF2 bootloader that allows firmware updates over USB.

Entering Bootloader

Method: Double-tap RESET pads quickly

Steps:

  1. Locate RESET pads on SIMux board (two small pads near MCU)

  2. Short the pads with tweezers, wire, or jumper twice quickly (< 0.5s between taps)

  3. Orange LED will fade (breathing effect)

  4. Device re-enumerates as bootloader

Bootloader Indicators:

  • Orange LED slowly pulsing (fade in/out)

  • Device appears as different COM port (Arduino bootloader)

Flashing on Windows

Option 1: BOSSA GUI (Easiest)

Download: SparkFun BOSSA GUI

Steps:

  1. Enter Bootloader:

    • Double-tap RESET pads on SIMux

    • Orange LED should fade in/out

  2. Check COM Port:

    • Open Device Manager (Win + X → Device Manager)

    • Expand Ports (COM & LPT)

    • Note the COM port (e.g., COM5)

    • Important: COM port in bootloader mode may differ from normal mode

  3. Launch BOSSA GUI:

    • Run BOSSA_GUI.exe

  4. Configure:

    • Binary: Click Browse, select firmware .bin file

    • Port: Select COM port from dropdown

    • Options: Check all 3 boxes:

      • Erase

      • Verify

      • SAMD21

  5. Flash:

    • Click Upload Binary

    • Progress bar shows upload status

    • Wait for completion (~10-20 seconds)

  6. Verify:

    • Device resets automatically

    • Orange LED stops fading (back to normal operation)

    • Connect serial terminal to verify new firmware

Option 2: BOSSA CLI

Download: BOSSA Releases

Steps:

  1. Download and extract BOSSA (e.g., bossa-1.9.1.zip)

  2. Enter Bootloader (double-tap RESET)

  3. Check COM Port:

    mode

    Or check Device Manager

  4. Open Command Prompt in BOSSA directory:

    cd path\to\bossa

  5. Flash Firmware:

    bossac.exe --port=COM5 -e -w -v -R path\to\firmware.bin

    Flags:

    • --port=COMx: Serial port (bootloader COM port)

    • -e: Erase flash before writing

    • -w: Write firmware

    • -v: Verify after writing

    • -R: Reset and run after flashing

  6. Wait for Completion:

    Erase flash
Write 65536 bytes to flash (512 pages)
[==============================] 100% (512/512 pages)
Verify 65536 bytes of flash
[==============================] 100% (512/512 pages)
Verify successful
Set boot flash true
CPU reset.

  7. Device Resets and runs new firmware

Flashing on Linux

Option 1: BOSSA CLI

Install:

Ubuntu/Debian:

sudo apt update
sudo apt install bossa-cli

Arch Linux:

sudo pacman -S bossa

From Source:

git clone https://github.com/shumatech/BOSSA.git
cd BOSSA
make
sudo make install

Steps:

  1. Enter Bootloader (double-tap RESET pads)

  2. Check Device:

    ls /dev/ttyACM*

    Example output: /dev/ttyACM0

    Note: Device path may change in bootloader mode (e.g., /dev/ttyACM1)

  3. Set Permissions (if needed):

    sudo usermod -a -G dialout $USER
# Logout and login for changes to take effect

    Or use sudo for the flashing command

  4. Flash Firmware:

    bossac --port=/dev/ttyACM0 -e -w -v -R firmware.bin

    Or with sudo:

    sudo bossac --port=/dev/ttyACM0 -e -w -v -R firmware.bin

  5. Verify Output:

    Erase flash
Write 65536 bytes to flash
[==============================] 100%
Verify 65536 bytes of flash
[==============================] 100%
Verify successful
CPU reset.

Common Issues:

Error: “No device found on /dev/ttyACM0”

  • Solution: Check device with ls /dev/tty*, bootloader may use different port

Error: “Device or resource busy”

  • Solution: Close any serial monitors or applications using the port

Verification

After flashing, verify firmware is running correctly:

1. Check Boot Sequence

  • Power LED lights up (green)

  • User LED blinks showing board count

  • SIM indicator LED lights for last selected SIM

2. Test Serial Connection

Open serial terminal (PuTTY, minicom, screen, etc.) at 115200 baud:

> INFO
MCU UID: E9E2FE2C8A0677068EFD0F787ABC1234
Serial Number: E9E2FE2C8A0677068EFD0F787ABC1234   # Should be the same as MCU UID (EEPROM cleared)
Boot Count: 1    # Should be 1 after first flash (EEPROM cleared)
...
