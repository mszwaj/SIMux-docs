Protocol Overview

SIMux supports two complementary control interfaces:

CLI (Command-Line Interface)

Human-friendly text commands sent over serial connection.

  • Format: Plain text, one command per line

  • Best for: Interactive use, manual testing, terminal access

  • Examples: INFO, STATE, SIM0=1

JSON

Machine-friendly structured protocol for automation and integration.

  • Format: JSON objects, one per line

  • Best for: Scripts, automation, programmatic control

  • Examples: {"cmd": "get_info"}, {"cmd": "set_sim", "board": 0, "sim": 1}

Note

Both interfaces work simultaneously. You can mix CLI commands and JSON in the same serial session when API is set to AUTO.

Serial Communication Settings

All communication uses standard USB CDC (Communications Device Class):

Parameter

Value

Baud rate

115200

Data bits

8

Stop bits

1

Parity

None

Flow control

None

Line ending

\n (LF) or \r\n (CRLF)

Connection Workflow

  1. Connect SIMux via USB-C to host

  2. Open serial port (e.g., COM3 on Windows, /dev/ttyACM0 on Linux)

  3. Configure terminal: 115200 baud, 8N1

  4. Send commands as text strings terminated with newline

  5. Receive responses as text

Response Behavior

CLI Commands

Success responses print informational text:

> INFO
DEVICE INFORMATION:
MCU UID: D1D522D429CA870AD5BFC3D71237B88D
Serial Number: SIMUX-DEV-01
Boot Count: 1
Current SIM states:
  Board 0 - 0x26: SIM 3

JSON API

All responses are JSON objects:

{
  "status": "ok",
  "cmd": "command_name",
  "data": { ... }
}

or

{
  "status": "error",
  "cmd": "command_name",
  "message": "Error description"
}

Always includes:

  • status: "ok" or "error"

  • cmd: Echo of the command sent

  • message: Human-readable status (on error or some success cases)

  • data: Response data (command-specific, optional)

Command Processing

Case Sensitivity

  • CLI commands: Case-insensitive (INFO = info = InFo)

  • JSON keys: Case-sensitive ("cmd" ≠ "CMD")

Error Handling

Malformed JSON

Returns parse error:

{
  "status": "error",
  "message": "JSON parse error"
}
