JSON API

Machine-friendly structured protocol for automation and integration.

Overview

The JSON API provides programmatic control over SIMux with structured request/response format.

Protocol: JSON over Serial (115200 baud)
Format: Single JSON object per line, terminated with \n

General Response Format

All responses follow this structure:

{
  "status": "ok|error",
  "cmd": "command_name",
  "message": "Human-readable status message",
  "data": { ... }
}

Fields:

  • status: "ok" for success, "error" for failure

  • cmd: Echo of the executed command

  • message: Informational message (optional, usually on errors or status changes)

  • data: Command-specific response data (optional)

SIM Control Commands

set_sim - Select SIM on Board

Select which SIM card slot to activate on a specific board.

Request:

{"cmd": "set_sim", "board": 0, "sim": 2}

Parameters:

  • board (number, required): Board index (0-7)

  • sim (number, required): SIM slot (0-4)

    • 0 = disable SIM

    • 1-4 = SIM slot number

Response (Success):

{
  "status": "ok",
  "cmd": "set_sim",
  "message": "Board 0 SIM set to 2",
  "data": {
    "board": 0,
    "address": "0x20",
    "sim": 2
  }
}

Response (Error - Invalid Board):

{
  "status": "error",
  "cmd": "set_sim",
  "message": "Board 3 not present",
  "data": {
    "board": 3,
    "board_count": 2
  }
}

Response (Error - Invalid SIM):

{
  "status": "error",
  "cmd": "set_sim",
  "message": "SIM value out of range (0-4)"
}

get_state - Get Current Device State

Returns current SIM selection and multiplexer state for all boards.

Request:

{"cmd": "get_state"}

Parameters: None

Response (Success):

{
  "status": "ok",
  "cmd": "get_state",
  "data": {
    "boards": [
      {
        "board": 0,
        "address": "0x20",
        "sim": 1,
        "mux2_en": true,
        "mux3_en": true,
        "mux_in": false
      },
      {
        "board": 1,
        "address": "0x21",
        "sim": 2,
        "mux2_en": false,
        "mux3_en": true,
        "mux_in": false
      }
    ],
    "board_count": 2
  }
}

Response Fields:

  • board_count: Number of detected boards

  • boards: Array of board states

    • board: Board number (0-7)

    • address: I2C address (hex string)

    • sim: Currently selected SIM (0-4)

    • mux2_en, mux3_en, mux_in: Multiplexer control signals (debug info)

System Information Commands

get_info - Get Device Information

Returns MCU UID, serial number, boot count, and board configuration.

Request:

{"cmd": "get_info"}

Parameters: None

Response (Success):

{
  "status": "ok",
  "cmd": "get_info",
  "data": {
    "mcu_uid": "D1D522D429CA870AD5BFC3D71237B88D",
    "serial_number": "SIMUX-DEV-01",
    "boot_count": 5,
    "board_count": 1,
    "boards": [
      {
        "board": 0,
        "address": "0x26",
        "sim": 2
      }
    ]
  }
}

Response Fields:

  • mcu_uid: Unique 128-bit MCU identifier (32 hex characters)

  • serial_number: Current serial number (MCU UID if not changed)

  • boot_count: Number of times device has booted

  • board_count: Number of boards detected on I2C bus

  • boards: Array of board information with current SIM selection

get_sn - Get Serial Number

Returns the current device serial number.

Request:

{"cmd": "get_sn"}

Parameters: None

Response (Success):

{
  "status": "ok",
  "cmd": "get_sn",
  "data": {
    "serial_number": "SIMUX-DEV-01"
  }
}

set_sn - Set Serial Number

Sets a new device serial number and triggers USB re-enumeration.

Request:

{"cmd": "set_sn", "value": "MY-DEVICE-1"}

Parameters:

  • value (string, required): New serial number (max 32-64 characters, no spaces)

Response (Success):

{
  "status": "ok",
  "cmd": "set_sn",
  "message": "Serial Number set. Resetting SIMux...",
  "data": {
    "serial_number": "MY-DEVICE-1"
  }
}

Response (Error):

{
  "status": "error",
  "cmd": "set_sn",
  "message": "Failed to set serial number. Must be 1-32 characters"
}

Side Effects:

  • Device will reconnect and re-enumerate on USB

  • Serial connection will be interrupted for ~2-3 seconds

  • Windows will detect on different COM port

  • Linux may detect as different ttyACM

clear_sn - Reset Serial Number to MCU UID

Restores the serial number to the MCU’s unique ID.

Request:

{"cmd": "clear_sn"}

Parameters: None

Response (Success):

{
  "status": "ok",
  "cmd": "clear_sn",
  "message": "Serial Number restored to default. Resetting SIMux...",
  "data": {
    "serial_number": "D1D522D429CA870AD5BFC3D71237B88D"
  }
}

Side Effects:

  • Device will reconnect and re-enumerate on USB

  • Serial connection will be interrupted for ~2-3 seconds

  • Windows will detect on different COM port

  • Linux may detect as different ttyACM

reset - Reset MCU

Triggers a microcontroller reset.

Request:

{"cmd": "reset"}

Parameters: None

Response:

{
  "status": "ok",
  "cmd": "reset",
  "message": "Resetting MCU..."
}

Side Effects:

  • Device will reset immediately

  • Serial connection will be interrupted

  • SIM states are preserved (loaded on boot)

Configuration Commands

autosave - Set Autosave Mode

Controls automatic saving of SIM state to flash.

Request (Change Mode):

{"cmd": "autosave", "value": "on"}

Parameters:

  • value (string, required):

    • "on": Enable auto-save after each SIM change (default)

    • "off": Disable auto-save (only save on reset)

    • "save": Immediately save all board states (doesn’t change mode)

Response (Mode Changed):

{
  "status": "ok",
  "cmd": "autosave",
  "message": "Autosave ON",
  "data": {
    "mode": "on"
  }
}

Response (Manual Save):

{
  "cmd": "autosave",
  "status": "ok",
  "message": "All boards saved"
}

Response (Error):

{
  "cmd": "autosave",
  "status": "error",
  "message": "Invalid value. Use: on, off, or save"
}

Notes:

  • Disabling autosave reduces flash wear when making many changes

  • State is always saved before reset, even if autosave is OFF

  • Mode is persistent

  • Power loss may cause no SIM states saved

api - Set API Mode

Changes the API mode to control input/output format.

Request:

{"cmd": "api", "value": "json"}

Parameters:

  • value (string, required): API mode

    • "auto": Auto-detect CLI vs JSON (default)

    • "json": Force JSON responses for all commands

    • "cli": Force CLI responses for all commands

Response (Success):

{
  "status": "ok",
  "cmd": "api",
  "message": "API mode changed",
  "data": {
    "mode": "CLI",
    "mode_code": 2
  }
}

Response (Error):

{
  "status": "error",
  "cmd": "api",
  "message": "Invalid mode. Use: auto, json, or cli"
}

Output Behavior:

Mode

Input Format

Output Format

auto

CLI command

CLI response

auto

JSON command

JSON response

json

JSON command

JSON response

json

CLI command

ERROR: CLI input not allowed in JSON mode

cli

CLI command

CLI response

cli

JSON command

ERROR: JSON input not allowed in CLI mode

Use Cases:

  • JSON mode: For programmatic control and parsing

  • CLI mode: For human-readable terminal interaction

  • AUTO mode: Flexibility to mix CLI and JSON commands

Notes:

  • Mode persists over power loss and reboots, saved to flash

  • Default mode is auto

Debug Commands

These commands are primarily for development, debugging and troubleshooting.

out - Toggle Output Route

Changes output routing (experimental feature).

Request:

{"cmd": "toggle_mux1"}

Parameters: None

Response:

{
  "status": "ok",
  "message": "Board 0 MUX1 toggled to HIGH",
  "cmd": "toggle_mux1",
  "data": {
    "board": 0,
    "state": true,
    "success": true
  }
}

Notes:

  • Internal debug feature to change SIM output to 2nd independent socket

  • Not typically needed for normal operation

i2c_scan - Scan I2C Bus

Manually scans I2C bus for connected boards and updates board count.

Request:

{"cmd": "i2c_scan"}

Parameters: None

Response:

{
  "status": "ok",
  "message": "Configuration unchanged - all OK",
  "cmd": "i2c_scan",
  "data": {
    "boards_found": 1,
    "boards": [
      {
        "address": "0x26"
      }
    ],
    "changed": false
  }
}

Response Fields:

  • board_found: Boards detected

  • address: Address of the board

  • changed: Whether board count changed

get_magic - Get Magic Number

Retrieves the EEPROM magic number for firmware/EEPROM compatibility verification.

Request:

{"cmd": "get_magic"}

Parameters: None

Response:

{
  "status": "ok",
  "cmd": "get_magic",
  "data": {
    "build_magic": "0xABCD1234",
    "stored_magic": "0xABCD1234",
    "match": true
  }
}

Response Fields:

  • build_magic: Magic number compiled into firmware

  • stored_magic: Magic number stored in EEPROM

  • match: Whether they match (compatibility check)

log - Set Logging

Enables or disables verbose debug logging.

Request:

{"cmd": "log", "value": "on"}

Parameters:

  • value (string, required): "on" or "off"

Response:

{
  "status": "ok",
  "cmd": "log",
  "message": "Logging DISABLED",
  "data": {
    "enabled": false
  }
}

Notes:

  • Changes serial output verbosity

  • Useful for troubleshooting

Usage Notes

Concurrent Use

JSON and CLI commands can be mixed in the same session (when API mode is auto):

{"cmd":"get_info"}       # JSON
INFO                     # CLI
{"cmd":"set_sim","board":0,"sim":1}  # JSON
1                        # CLI

Both produce equivalent results.

State Persistence

  • Data to persist across power cycles and RESET command:

    • SIMs set on all connected boards

    • boot count

    • serial number

    • magic number

    • API mode

    • autosave mode

Case Sensitivity

  • Command names (cmd field) are case-sensitive in JSON

  • Use lowercase: "get_info" not "GET_INFO"
