CLI Commands

Human-friendly text commands for interactive terminal control.

SIM Control Commands

Select SIM (Quick - Main Board Only)

Syntax: <0-4>

Description: Select SIM state for main board (board 0) using single digit.

Command

Effect

0

Disable SIM

1

Activate SIM1

2

Activate SIM2

3

Activate SIM3

4

Activate SIM4

Examples:

1
Board 0 SIM set to 1

0
Board 0 SIM set to 0

Notes:

  • Works only for main board (board 0)

  • LED indicator shows selected SIM

Select SIM (Specific Board)

Syntax: SIM<board>=<number>

Description: Select SIM for any board in multi-board configuration.

Parameters:

  • <board>: Board number (0-7)

  • <number>: SIM number (0-4)

    • 0 = SIM disabled

    • 1-4 = SIM slot number

Examples:

SIM0=1
Board 0 SIM set to 1

SIM1=3
Board 1 SIM set to 3

SIM2=0
Board 2 SIM set to 0

Notes:

  • Required for extension boards (board 1-7)

  • SIM0=x is equivalent to just typing x

Get State

Syntax: STATE

Description: Display current SIM selection for all detected boards with their addresses.

Output Example (2 boards):

CURRENT STATE...
Board 0 @0x26 SIM: 1
MUX2 EN: 0 | MUX3 EN: 1 | IN: 0

Output Fields:

  • Board number and I2C address

  • Currently selected SIM slot

  • Multiplexer pin states

System Information Commands

Get Device Info

Syntax: INFO

Description: Display comprehensive device information.

Output Example:

INFO
MCU UID: E9E2FE2C8A0677068EFD0F787ABC1234
Serial Number: SIMUX-DEV-01
Boot Count: 127
Current SIM states:
  Board 0 - 0x26: SIM 0
  Board 1 - 0x27: SIM 3

Output Fields:

  • MCU UID: Unique 128-bit hardware identifier

  • Serial Number: Current serial number (default: MCU UID)

  • Boot Count: Number of times device has booted

  • Boards Detected: Number of boards on I2C bus

  • Board Status: Current SIM selection for each board

Get Serial Number

Syntax: GETSN

Description: Display only the device serial number.

Output Example:

GETSN
Serial Number: SIMUX-DEV-01

Notes:

  • Default serial number is MCU UID

  • Can be customized with SETSN command

Set Serial Number

Syntax: SETSN=<name>

Description: Assign a custom serial number to the device.

Parameters:

  • <name>: Alphanumeric string with up to 32 chars (no spaces)

Examples:

SETSN=SIMUX-DEV-01
Serial number set to: SIMUX-DEV-01

SETSN=DEV_BENCH_A
Serial number set to: DEV_BENCH_A

Notes:

  • Useful for identifying multiple devices

  • Persists across power cycles, resets and different host devices

  • Maximum length: 32 characters

  • Spaces not allowed - use underscores or hyphens

Clear Serial Number

Syntax: CLEARSN

Description: Reset serial number to default (MCU UID).

Output Example:

CLEARSN
Serial Number restored to default
Resetting SIMux for new SN...

Reset Device

Syntax: RESET

Description: Reboot the SIMux controller.

Output Example:

RESET
RESETting MCU...

Notes:

  • SIM state is preserved (stored in non-volatile memory)

  • Device will disconnect and reconnect on serial port

  • Boot count will be incremented during boot

Help

Syntax: HELP or ?

Description: Display list of available commands.

Output Example:

HELP
AVAILABLE COMMANDS:
0-4          : Select SIM (0=off, 1-4=SIM slots) on first board
SIMx=0-4     : Select SIM on board x (0-7)
STATE        : Show current status for all boards
INFO         : Show device information
GETSN        : Get serial number
SETSN=x      : Set serial number
CLEARSN      : Restore serial number to MCU UID (default)
AUTOSAVE=x   : Set autosave mode (ON/OFF/SAVE)
RESET        : Reset MCU
HELP/?       : Show this help
API=mode     : Set API mode (AUTO/JSON/CLI)
OUT          : Toggle output (MUX1)
I2C          : Scan I2C for boards
MAGIC        : Show EEPROM magic number
LOG          : Toggle logging on/off

Set API Mode

Syntax: API=<mode>

Description: Switch between CLI and JSON input and output formats.

Parameters:

  • <mode>: API mode selector

    • AUTO - Auto-detect format based on input (default)

    • JSON - Force JSON input and output

    • CLI - Force human-readable CLI input and output

Examples:

API=JSON
API mode set to: JSON

API=CLI
API mode set to: CLI

API=AUTO
API mode set to: AUTO

Output Behavior:

Mode

Input Format

Output Format

AUTO

CLI command

CLI response

AUTO

JSON command

JSON response

JSON

JSON command

JSON response

JSON

CLI command

ERROR: CLI input not allowed in JSON mode

CLI

CLI command

CLI response

CLI

JSON command

ERROR: JSON input not allowed in CLI mode

Use Cases:

  • JSON mode: For programmatic control and parsing

  • CLI mode: For human-readable terminal interaction

  • AUTO mode: Flexibility to mix CLI and JSON commands

Notes:

  • Mode persists over power loss and reboots, saved to flash

  • Default mode is AUTO

  • See JSON API for JSON command format

Debug Commands

These commands are primarily for development, debugging and troubleshooting.

Output Route

Syntax: OUT

Description: Change output routing (experimental feature).

Notes:

  • Internal debug feature to change SIM output to 2nd independent socket

  • Not typically needed for normal operation

I2C Bus Scan

Syntax: I2C

Description: Manually scan I2C bus and detect connected boards.

Output Example:

I2C
I2C Scan Results:
  Boards before: 1
  Boards after:  1
  Found: 1
  Addresses: 0x26
Configuration unchanged

Use Cases:

  • Verify extension boards are detected

  • Troubleshoot I2C connection issues

  • Detect address changes in runtime

Magic Number

Syntax: MAGIC

Description: Display build-time and EEPROM magic numbers.

Output Example:

MAGIC
Build Magic: 0xABCD1234
Stored Magic: 0xABCD1234
Match: Yes

Use Cases:

  • Verify firmware/EEPROM compatibility

  • Diagnose firmware update issues

  • Development and debugging

Toggle Logging

Syntax: LOG

Description: Enable or disable verbose debug logging.

Output Example:

LOG
Logging DISABLED

LOG
Logging ENABLED

Notes:

  • Increases serial output verbosity

  • Useful for troubleshooting

Command Behavior Notes

Case Sensitivity

  • All CLI commands are case-insensitive

  • INFO = info = InFo

Whitespace

  • Leading and trailing whitespace is ignored

  • INFO works the same as INFO

Invalid Commands

  • Unknown commands will return output “Unknown command”

State Persistence

  • Data to persist across power cycles and RESET command:

    • SIMs set on all connected boards

    • boot count

    • serial number

    • magic number

    • API mode

    • autosave mode
