System Architecture

High-level system design and signal flow for SIMux.

Overview

SIMux uses a clean separation between control logic and SIM signal routing:

  • Control Circuit: MCU, button, LEDs, I2C bus

  • SIM Circuit: Passive analog switches (multiplexers)

This architecture ensures that control decisions never interfere with SIM protocol timing, maintaining full ISO/IEC 7816-3 compliance.

Control Circuit

MCU (Seeeduino XIAO SAMD21)

Role: Main controller and USB interface

Responsibilities:

  • USB CDC communication with host (115200 baud virtual serial)

  • Command parsing (CLI and JSON)

  • I2C master for GPIO expanders

  • Button debouncing

  • State persistence (EEPROM/Flash)

  • Boot count and serial number management

Key Features:

  • ARM Cortex-M0+ @ 48 MHz

  • 256 KB Flash, 32 KB SRAM

  • Native USB support

  • I2C, UART peripherals

  • Custom serial number support (via forked Arduino core)

GPIO Expander (MCP23008)

Role: I2C-controlled I/O expansion for each board

Responsibilities:

  • Control multiplexers select/enable lines

  • Read button state

  • Drive LED indicators

  • Provide unique I2C address per board (0x20-0x27)

Why MCP23008?

  • Simple I2C interface

  • 8 I/O pins sufficient for board needs

  • Hardware address pins enable multi-board addressing

  • Low cost and widely available

Button

Role: Manual SIM selection without host

Behavior:

  • Each press increments to next SIM state (0→1→2→3→4→0)

  • Debounced in firmware (50ms)

  • Works simultaneously with serial commands

  • Button action logged to serial terminal if connected

LEDs

Types:

  • Power LED (green): Always on when powered

  • RX/TX LEDs (blue): Serial communication activity

  • User LED (orange): Boot count indicator, SIM change feedback

  • SIM Indicator LEDs (1-4): Show which SIM is active

See LED Indicators for complete reference.

SIM Signal Circuit

Analog Multiplexers

Role: Route SIM signals from selected slot to FPC output

Architecture:

  • Three ICs per board (6 signal lines total)

    • Two for selecting 1 out of 4 SIM slots

    • One for selecting output route (experimental)

Signal Lines Switched:

  1. VCC (SIM power)

  2. RST (Reset)

  3. CLK (Clock)

  4. GND (Ground)

  5. VPP (Programming voltage)

  6. I/O (Data)

Key Characteristics:

  • Low on-resistance (~5Ω): Minimal signal distortion

  • Passive: No active signal modification

SIM Card Slots

Type: nanoSIM (4FF) push-push sockets

Features:

  • Spring-loaded ejection mechanism

  • Supports standard nanoSIM cards (1.8V or 3V)

  • Card Detection (not in use at the momment)

FPC Connector

Type: 6-pin, 1mm pitch, bottom contact

Purpose: Connects multiplexed SIM signals to target device

Pinout: Standard ISO 7816 (VCC, RST, CLK, GND, VPP, I/O)

Cable: Flexible Printed Circuit (FPC) cable routes signals to target device’s SIM slot

See FPC Compatibility Matrix for target device connections.

Address Selection: Hardware address pins (A0, A1, A2) on MCP23008 determine address by switching DIP switch.

Bus Topology

Physical: Daisy-chain via Grove connectors

[MCU] ←→ [Grove] ←→ [Board 0 Expander] ←→ [Grove] ←→ [Board 1 Expander] ←→ ...

Electrical: Multi-drop bus (all devices in parallel) even though board to board is a daisy-chain connection

Speed: 100 kHz (I2C Standard Mode)

Pull-ups: 4.7kΩ resistors on main board (SCL, SDA) realized by jumpers (from hw 0.7.0)

Power Distribution

Main Board

Input: 5V DC from USB-C connector

Regulation:

  • XIAO SAMD21 has onboard 3.3V regulator

  • 3.3V supplied to:

    • MCP23008

    • multiplexers

    • LEDs (via current-limiting resistors)

    • I2C pull-ups

    • Grove connectors (VCC pin)

Extension Boards

Power Source: 3.3V from I2C Grove cable

Components Powered:

  • MCP23008

  • multiplexers

  • LEDs

SIM Card Power

Source: Target device (DUT) via FPC cable

SIMux does NOT power SIM cards - they are powered by the target device through the multiplexer. SIMux only switches which SIM card’s power line connects to the target.

ISO/IEC 7816-3 Compliance

No Timing Modification

Key Design Principle: SIM signals are passively switched, not actively driven or buffered.

  • Multiplexers are analog switches (not digital buffers)

  • No clock division or phase shifting

  • No protocol state machines in signal path

  • Rise/fall times preserved (determined by target device and SIM card)

Signal Integrity

  • Low on-resistance (~5Ω) minimizes voltage drop

  • Short PCB traces reduce parasitic capacitance

  • Ground plane reduces noise

For automated testing, verify target device supports SIM hot-swapping before enabling automated switching.

Design Principles

  1. Separation of Concerns: Control logic and SIM signals are electrically isolated

  2. Passive Multiplexing: No active signal modification ensures protocol compliance

  3. Modularity: Identical boards enable easy extension

  4. Persistence: State survives power loss (emulated EEPROM storage)

  5. Feedback: LEDs provide visual feedback; serial provides responses and verbose logging
