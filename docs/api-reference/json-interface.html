JSON Interface

Overview

The JSON Interface provides programmatic control over SIMux with structured request/response format.

Protocol: JSON over Serial (115200 baud)
Format: Single JSON object per line, terminated with \n

General Response Format

All responses follow this structure:

{
  "status": "ok|error",
  "cmd": "command_name",
  "message": "Human-readable status message",
  "data": { ... }
}

Fields:

  • status: "ok" for success, "error" for failure

  • cmd: Echo of the executed command

  • message: Informational message (optional, usually on errors or status changes)

  • Additional command-specific fields (optional, flattened at top level)

SIM Control Commands

set_sim - Select SIM on Board

Select which SIM card slot to activate on a specific board.

Request:

{"cmd": "set_sim", "board": "0x20", "sim": 2}

Parameters:

  • board (string, required): Board address hex-string (0x20-0x27)

  • sim (number, required): SIM slot (0-4)

    • 0 = disable SIM

    • 1-4 = SIM slot number

Response (Success):

{
  "status": "ok",
  "cmd": "set_sim",
  "message": "Board 0x20 SIM set to 2",
  "board": "0x20",
  "sim": 2
}

Response (Error - Invalid Board):

{
  "status": "error",
  "cmd": "set_sim",
  "message": "Board 0x21 not present",
  "board": "0x21"
}

Response (Error - Invalid SIM):

{
  "status": "error",
  "cmd": "set_sim",
  "message": "Invalid SIM value (must be 0-4)",
  "board": "0x22"
}

get_state - Get Current Device State

Returns current SIM selection and multiplexer state for all boards.

Request:

{"cmd": "get_state"}

Parameters: None

Response (Success):

{
  "status": "ok",
  "cmd": "get_state",
  "boards": [
    {
      "board": "0x20",
      "sim": 4
    },
    {
      "board": "0x22",
      "sim": 3
    }
  ],
  "board_count": 2
}

Response Fields:

  • board_count: Number of detected boards

  • boards: Array of board states

    • board: I2C address (hex string)

    • sim: Currently selected SIM (0-4)

Note

In debug firmware builds, each board entry also includes:

  • out: MUX1 channel ("a" or "b")

  • mux2_en, mux3_en, mux_in: Multiplexer control signals

System Information Commands

get_info - Get Device Information

Returns MCU UID, serial number, boot count, and board configuration.

Request:

{"cmd": "get_info"}

Parameters: None

Response (Success):

{
  "status": "ok",
  "cmd": "get_info",
  "mcu_uid": "D1D522D429CA870AD5BFC3D71237B88D",
  "serial_number": "D1D522D429CA870AD5BFC3D71237B88D",
  "boot_count": 34,
  "board_count": 2,
  "boards": [
    {
      "board": "0x20",
      "sim": 4
    },
    {
      "board": "0x22",
      "sim": 3
    }
  ]
}

Response Fields:

  • mcu_uid: Unique 128-bit MCU identifier (32 hex characters)

  • serial_number: Current serial number (MCU UID if not changed)

  • boot_count: Number of times device has booted

  • board_count: Number of boards detected on I2C bus

  • boards: Array of board information with current SIM selection

get_sn - Get Serial Number

Returns the current device serial number.

Request:

{"cmd": "get_sn"}

Parameters: None

Response (Success):

{
  "status": "ok",
  "cmd": "get_sn",
  "serial_number": "D1D522D429CA870AD5BFC3D71237B88D"
}

set_sn - Set Serial Number

Sets a new device serial number and triggers USB re-enumeration.

Request:

{"cmd": "set_sn", "value": "MY-DEVICE-1"}

Parameters:

  • value (string, required): New serial number (max 32 characters, no spaces)

Response (Success):

{
  "status": "ok",
  "cmd": "get_sn",
  "serial_number": "D1D522D429CA870AD5BFC3D71237B88D"
}

Response (Error):

{
  "status": "error",
  "cmd": "set_sn",
  "message": "Failed to set serial number. Must be 1-32 characters"
}

Side Effects:

  • Device will reconnect and re-enumerate on USB

  • Serial connection will be interrupted for ~2-3 seconds

  • Windows will detect on different COM port

  • Linux may detect as different ttyACM

clear_sn - Reset Serial Number to MCU UID

Restores the serial number to the MCU’s unique ID.

Request:

{"cmd": "clear_sn"}

Parameters: None

Response (Success):

{
  "status": "ok",
  "cmd": "clear_sn",
  "message": "Serial Number restored to default. Rebooting SIMux...",
  "serial_number": "D1D522D429CA870AD5BFC3D71237B88D"
}

Side Effects:

  • Device will reconnect and re-enumerate on USB

  • Serial connection will be interrupted for ~2-3 seconds

  • Windows will detect on different COM port

  • Linux may detect as different ttyACM

reboot - Reboot MCU

Triggers a microcontroller reboot.

Request:

{"cmd": "reboot"}

Parameters: None

Response:

{
  "status": "ok",
  "cmd": "reboot",
  "message": "Rebooting MCU..."
}

Side Effects:

  • Device will reboot immediately

  • Serial connection will be interrupted

  • SIM states are preserved (loaded on boot)

Configuration Commands

set_autosave - Set Autosave Mode

Controls automatic saving of SIM state to flash.

Request (Change Mode):

{"cmd": "set_autosave", "value": "on"}

Parameters:

  • value (string, required):

    • "on": Enable auto-save after each SIM change (default)

    • "off": Disable auto-save

Response (Mode Changed):

{
  "status": "ok",
  "cmd": "set_autosave",
  "message": "Autosave ON",
  "mode": "on"
}

Response (Error):

{
  "status": "error",
  "cmd": "set_autosave",
  "message": "Invalid value. Use: on or off"
}

Notes:

  • Disabling autosave reduces flash wear when making many changes

  • State is always saved before reboot, even if autosave is OFF

  • Mode is persistent

  • Power loss may cause no SIM states saved

save_state - Save State to Flash

Immediately saves the current SIM state of all boards to flash without changing autosave mode.

Request:

{"cmd": "save_state"}

Parameters: None

Response:

{
  "status": "ok",
  "cmd": "save_state",
  "message": "All boards saved"
}

Notes:

  • Useful when autosave is OFF and you want to manually persist the current state

set_api - Set API Mode

Changes the API mode to control input/output format.

Request:

{"cmd": "set_api", "value": "json"}

Parameters:

  • value (string, required): API mode

    • "auto": Auto-detect Text Command Interface vs JSON Interface (default)

    • "json": Force JSON responses for all commands

    • "cli": Force Text Command Interface responses for all commands

Response (Success):

{
  "status": "ok",
  "cmd": "set_api",
  "message": "API mode changed",
  "mode": "JSON"
}

Response (Error):

{
  "status": "error",
  "cmd": "set_api",
  "message": "Invalid mode. Use: auto, json, or cli"
}

Output Behavior:

Mode

Input Format

Output Format

auto

Text Command Interface command

Text Command Interface response

auto

JSON command

JSON response

json

JSON command

JSON response

json

Text Command Interface command

ERROR: Text Command Interface input not allowed in JSON mode

cli

Text Command Interface command

Text Command Interface response

cli

JSON command

ERROR: JSON input not allowed in Text Command Interface mode

Use Cases:

  • JSON mode: For programmatic control and parsing

  • Text Command Interface mode: For human-readable terminal interaction

  • AUTO mode: Flexibility to mix Text Command Interface and JSON Interface commands

Notes:

  • Mode persists over power loss and reboots, saved to flash

  • Default mode is auto

scan_i2c - Scan I2C Bus

Manually scans I2C bus for connected boards and updates board count.

Request:

{"cmd": "scan_i2c"}

Parameters: None

Response:

{
  "status": "ok",
  "message": "Configuration unchanged - all OK",
  "cmd": "scan_i2c",
  "boards_found": 2,
  "boards": [
    "0x20",
    "0x22"
  ],
  "changed": false
}

Response Fields:

  • boards_found: Boards detected

  • boards: Array of found boards with listed addresses

  • changed: Whether board count changed in runtime

Debug Commands

These commands are primarily for development, debugging and troubleshooting.

set_out - Set Output Channel

Sets the MUX1 output channel on a specified board.

Request:

{"cmd": "set_out", "board": "0x20", "channel": "a"}

Parameters:

  • board (hex string, optional): Board index (default: 0x20)

  • channel (string, required): "a" or "b"

Response (Success):

{
  "status": "ok",
  "message": "Board 0x20 MUX1 set to channel a",
  "cmd": "set_out",
  "board": "0x20",
  "channel": "a"
}

Response (Error - Invalid Channel):

{
  "status": "error",
  "cmd": "set_out",
  "message": "Invalid channel. Use: a or b"
}

Notes:

  • Internal debug feature to change SIM output to 2nd independent socket

  • Not typically needed for normal operation

get_magic - Get Magic Number

Retrieves the EEPROM magic number for firmware/EEPROM compatibility verification.

Request:

{"cmd": "get_magic"}

Parameters: None

Response:

{
  "status": "ok",
  "cmd": "get_magic",
  "build_magic": "0xABCD1234",
  "stored_magic": "0xABCD1234",
  "match": true
}

Response Fields:

  • build_magic: Magic number compiled into firmware

  • stored_magic: Magic number stored in EEPROM

  • match: Whether they match (compatibility check)

set_log - Set Logging

Enables or disables verbose debug logging explicitly.

Request:

{"cmd": "set_log", "value": "off"}

Parameters:

  • value (string, required): "on" or "off"

Response:

{
  "status": "ok",
  "cmd": "set_log",
  "message": "Logging DISABLED",
  "enabled": false
}

Notes:

  • Logging state is saved to flash and persists across reboot and power cycle.

  • Changes serial output verbosity

  • Useful for troubleshooting

get_flash_dump - Dump Flash Data

Returns the raw contents of the flash MyData structure.

Request:

{"cmd": "get_flash_dump"}

Parameters: None

Response:

{
  "status": "ok",
  "cmd": "get_flash_dump",
  "currentSIMs_stored": [4, 1, 3, 1, 1, 1, 1, 1],
  "detected_boards": ["0x20", "0x22"],
  "bootCount": 35,
  "sn": "D1D522D429CA870AD5BFC3D71237B88D",
  "magicNumber": "0xABCD1234",
  "apiMode": 0,
  "apiMode_name": "AUTO",
  "autosaveMode": 0,
  "autosaveMode_name": "ON",
  "loggingEnabled": 0,
  "loggingEnabled_name": "DISABLED"
}

Notes:

  • Shows raw values stored in flash memory

  • Useful for debugging persistence issues

Usage Notes

Concurrent Use

JSON Interface and Text Command Interface can be mixed in the same session (when API mode is auto):

{"cmd":"get_info"}       # JSON
INFO                     # Text Command Interface
{"cmd":"set_sim","board":0,"sim":1}  # JSON
1                        # Text Command Interface

Both produce equivalent results.

State Persistence

  • Data to persist across power cycles and REBOOT:

    • SIMs set on all connected boards

    • boot count

    • serial number

    • magic number

    • API mode

    • autosave mode

    • logging mode (enabled/disabled)

Case Sensitivity

  • Command names (cmd field) are case-sensitive in JSON

  • Use lowercase: "get_info" not "GET_INFO"
